This guide walks through creating a custom domain module that follows the framework's conventions. The pattern is derived from the `organization` and `compliance` modules.

## Overview

A module is a class that implements the `Module` interface and packages domain workflows, database schemas, validation schemas, and event definitions. The framework calls `initialize(units)` to inject dependencies, `prepare()` to set up schema/handlers, and `destroy()` to clean up.

## File Structure

```
packages/my-module/
  src/
    index.ts              # Module class, type re-exports, dbSchema export
    db-schema.ts          # Drizzle pgTable/pgEnum definitions
    types.ts              # Type re-exports from schemas + domain interfaces
    event-map.ts          # Event constants + typed event interfaces + EventMap type
    constants.ts          # Module-specific enums (as const objects) [optional]
    schemas/
      index.ts            # Re-exports all schemas + types
      enums.ts            # Valibot enum schemas mirroring constants
      utils.ts            # Shared valibot schema utilities (regex, lengths)
      entity.ts           # Per-entity valibot schemas
    workflows/
      entity.ts           # Business logic classes
  package.json
  tsconfig.json
```

## Step 1: Package Setup

`package.json`:

```json
{
  "name": "@aspen-os/my-module",
  "type": "module",
  "exports": { ".": "./src/index.ts" },
  "dependencies": {
    "@aspen-os/framework": "workspace:*",
    "@aspen-os/constants": "workspace:*",
    "drizzle-orm": "catalog:",
    "valibot": "catalog:"
  }
}
```

<Callout type="info">
  No build step. Package `exports` point at raw `.ts` files. Bun resolves them directly.
</Callout>

## Step 2: Database Schema

Create `src/db-schema.ts` with Drizzle table and enum definitions:

```ts
import { relations } from "drizzle-orm";
import {
  boolean,
  index,
  jsonb,
  pgEnum,
  pgTable,
  text,
  timestamp,
} from "drizzle-orm/pg-core";

// Enums reference constant objects when available
export const statusEnum = pgEnum("my_entity_status", ["active", "inactive"]);

export const myEntity = pgTable(
  "my_entity",
  {
    id: text("id").defaultRandom().primaryKey(),
    name: text("name").notNull(),
    status: statusEnum().default("active").notNull(),
    metadata: jsonb("metadata").default({}),
    createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
    updatedAt: timestamp("updated_at", { withTimezone: true }).notNull().defaultNow(),
  },
  (table) => [index("idx_my_entity_status").on(table.status)],
);

export const myEntityRelations = relations(myEntity, ({ many }) => ({}));
```

<Callout type="warn">
  IDs are always `text` with `DEFAULT gen_random_uuid()::text` — never native UUID columns. Timestamps use `TIMESTAMPTZ` / `withTimezone: true`.
</Callout>

## Step 3: Constants & Enums

`src/constants.ts`:

```ts
export const MY_ENTITY_STATUS = {
  ACTIVE: "active",
  INACTIVE: "inactive",
} as const;

export type MyEntityStatus =
  (typeof MY_ENTITY_STATUS)[keyof typeof MY_ENTITY_STATUS];
```

## Step 4: Validation Schemas (Valibot)

`src/schemas/entity.ts`:

```ts
import { type InferOutput, object, optional, pipe, string, minLength } from "valibot";

export const CreateMyEntitySchema = object({
  name: pipe(string(), minLength(1, "Name is required")),
  status: optional(string()),
});

export type CreateMyEntityInput = InferOutput<typeof CreateMyEntitySchema>;

export const UpdateMyEntitySchema = object({
  name: optional(string()),
  status: optional(string()),
});

export type UpdateMyEntityInput = InferOutput<typeof UpdateMyEntitySchema>;

export const MyEntityFiltersSchema = object({
  status: optional(string()),
});

export type MyEntityFilters = InferOutput<typeof MyEntityFiltersSchema>;
```

## Step 5: Event Map

`src/event-map.ts`:

```ts
export const MY_ENTITY_EVENTS = {
  CREATED: "my-module:entity_created",
  UPDATED: "my-module:entity_updated",
} as const;

export interface MyEntityCreatedEvent {
  entity: { id: string; name: string };
}

export interface MyEntityUpdatedEvent {
  changes: Record<string, unknown>;
  entity: { id: string; name: string };
}

export type MyModuleEventMap = {
  [MY_ENTITY_EVENTS.CREATED]: MyEntityCreatedEvent;
  [MY_ENTITY_EVENTS.UPDATED]: MyEntityUpdatedEvent;
};
```

## Step 6: Workflow

`src/workflows/entity.ts`:

```ts
import type { NodePgDatabase } from "drizzle-orm/node-postgres";
import { eq } from "drizzle-orm";
import { parse } from "valibot";

import { myEntity } from "../db-schema";
import { CreateMyEntitySchema, type CreateMyEntityInput } from "../schemas/entity";

export class MyEntityWorkflow {
  constructor(private readonly db: NodePgDatabase) {}

  async create(input: CreateMyEntityInput) {
    const data = parse(CreateMyEntitySchema, input);
    const [result] = await this.db
      .insert(myEntity)
      .values(data)
      .returning();
    return result;
  }

  async get(id: string) {
    const [result] = await this.db
      .select()
      .from(myEntity)
      .where(eq(myEntity.id, id));
    return result ?? null;
  }

  async list() {
    return this.db.select().from(myEntity);
  }
}
```

## Step 7: Module Class

`src/index.ts`:

```ts
import type { DatabaseUnit, PubSubUnit } from "@aspenos/framework/server";

import * as dbSchema from "./db-schema";
import { MyEntityWorkflow } from "./workflows/entity";

export type { MyEntityStatus } from "./constants";
export type {
  CreateMyEntityInput,
  MyEntityFilters,
  UpdateMyEntityInput,
} from "./types";
export { dbSchema };

export type MyModuleConfig = {
  // module-specific config options
};

export class MyModule {
  static create(config: MyModuleConfig): MyModule {
    return new MyModule(config);
  }

  constructor(private config: MyModuleConfig) {}

  readonly db_schema = dbSchema;
  readonly name = "my-module";

  #entities: MyEntityWorkflow | null = null;

  get entities(): MyEntityWorkflow {
    if (!this.#entities) throw notInitialized();
    return this.#entities;
  }

  initialize(units: { db: DatabaseUnit; pubsub: PubSubUnit }): void {
    this.#entities = new MyEntityWorkflow(units.db.db);
  }

  async prepare(): Promise<void> {
    const { pushSchema } = await import("drizzle-kit/api");
    const result = await pushSchema(dbSchema, units.db.db);
    if (result.statementsToExecute.length > 0) {
      console.log("[my-module] Applying schema");
      await result.apply();
    }
  }

  async destroy(): Promise<void> {
    this.#entities = null;
  }
}

function notInitialized(): Error {
  return new Error(
    "My module not initialized. Call initialize() after framework.create().",
  );
}
```

## Step 8: Use It

```ts
import { Framework } from "@aspen-os/framework/server";
import { MyModule } from "@aspen-os/my-module";

const myModule = MyModule.create({});

const framework = Framework.create(config, {
  "my-module": myModule,
});

await framework.prepare();

await framework.run(async () => {
  const entity = await framework["my-module"].entities.create({
    name: "My Entity",
  });
});
```

## Key Conventions

| Convention | Rule |
|---|---|
| Static `create(config)` factory | Always use a static factory, not `new` directly |
| `#` prefix for private fields | `#entities: MyEntityWorkflow \| null = null` |
| Lazy getters with `notInitialized()` | Throw if accessed before `initialize()` |
| `db_schema` export | The drizzle schema namespace |
| `name` as kebab-case | `readonly name = "my-module"` |
| `prepare()` pushes schema | Uses `pushSchema()` from `drizzle-kit/api` |
| `destroy()` nulls fields | Unregisters handlers and nulls private fields |
| Valibot for validation | Domain module input validation |
| Events as `"domain:event_name"` | Typed via `EventMap`, published as plain strings |
