## Entry Surfaces

The framework package (`@aspen-os/framework`) declares only `./client` and `./server` in its `exports` map. There is no root `.` entry in the package.json ‚Äî the bare import `@aspenos/framework` works via Bun workspace resolution.

### Server (`@aspen-os/framework/server`)

Exports:
- `Framework` class (with static `create()` factory)
- `FrameworkConfig`, `FrameworkInstance` types
- `Unit`, `Module` interfaces
- All 7 unit classes and their config types:
  - `DatabaseUnit`, `DatabaseConfig`
  - `AuthUnit`, `AuthConfig`
  - `LogUnit`, `LogConfig`
  - `PubSubUnit`, `PubSubConfig`
  - `StorageUnit`, `StorageConfig`
  - `RpcUnit`, `RpcConfig`
  - `KvStoreUnit`, `KvStoreConfig`

### Client (`@aspen-os/framework/client`)

Exports:
- Client `Framework` class (browser-side, 3 units only)
- `createAccessControl` (re-exported from `better-auth/plugins/access`)
- Client config types

The client framework has 3 units:
- `AuthUnit` ‚Äî wraps better-auth React client
- `LogUnit` ‚Äî stub (throws on `prepare()`/`destroy()`)
- `RpcUnit` ‚Äî stub (no-op)

## FrameworkConfig

All 7 units are **required** in the config:

```ts
type FrameworkConfig = {
  auth: AuthConfig;
  db: DatabaseConfig;
  kvStore: KvStoreConfig;
  logs: LogConfig;
  pubsub: PubSubConfig;
  rpc: RpcConfig;
  storage: StorageConfig;
};
```

## Lifecycle

```text
Framework.create(config, modules)   ‚Üí  creates units + initializes modules
    ‚Üì
framework.prepare()                 ‚Üí  unit.$prepare() then module.prepare()
    ‚Üì
framework.run(fn)                   ‚Üí  AsyncLocalStorage { db, pubsub }
    ‚Üì
framework.destroy()                 ‚Üí  module.destroy() then unit.$destroy()
```

### `create(config, modules)`

Static factory ‚Äî the **only** way to construct a Framework. It:
1. Instantiates all 7 units in dependency order
2. Calls `module.initialize(units)` on each module
3. Returns a proxy-wrapped `FrameworkInstance`

<Callout type="warn">
  There is no `new Framework(config)` or `registerModule()`. All modules must be passed to `create()` at once.
</Callout>

### `prepare()`

Runs each unit's `$prepare()` (e.g. `DatabaseUnit.$prepare()` calls `pushSchema()`), then each module's `prepare()`. Errors are caught and logged per-unit/module.

### `run(fn)`

Executes `fn` inside `AsyncLocalStorage` providing `{ db, pubsub }`:
- `db` ‚Äî the drizzle `NodePgDatabase`
- `pubsub` ‚Äî the `PubSubUnit`

### `destroy()`

Runs module `destroy()` then unit `$destroy()`. Errors are caught and logged.

## Dependency Graph

Units are constructor-injected:

```text
DatabaseUnit ‚Üê LogUnit
DatabaseUnit ‚Üê PubSubUnit
DatabaseUnit ‚Üê StorageUnit
DatabaseUnit ‚Üê AuthUnit
DatabaseUnit ‚Üê RpcUnit (also ‚Üê LogUnit, PubSubUnit, AuthUnit)
DatabaseUnit ‚Üê KvStoreUnit
```

| Unit | Injected deps |
|---|---|
| `db` | ‚Äî (owns `pg.Pool` + drizzle `db`) |
| `logs` | `{ db }` |
| `pubsub` | `{ db }` |
| `storage` | `{ db }` |
| `auth` | `{ db }` |
| `rpc` | `{ auth, db, logs, pubsub }` |
| `kvStore` | `{ db }` |

## Accessing Units and Modules

The `FrameworkInstance` is a Proxy that intercepts property access:

```ts
// Proxy access (preferred for readability)
framework.db              // ‚Üí DatabaseUnit
framework.auth             // ‚Üí AuthUnit
framework.organization     // ‚Üí OrganizationModule

// Typed accessor methods
framework.getUnit("db")        // ‚Üí DatabaseUnit
framework.getModule("organization") // ‚Üí OrganizationModule (throws if not found)
```
