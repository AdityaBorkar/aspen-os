The `RpcUnit` (`$name: "rpc"`) provides an RPC layer using [oRPC](https://orpc.uno) with Zod input validation. It exposes a router and an HTTP handler.

## Configuration

```ts
interface RpcConfig {
  prefix?: string;  // default: "/api/rpc"
}
```

### Example

```ts
const rpcConfig: RpcConfig = {
  prefix: "/api/rpc",
};
```

## Public API

```ts
class RpcUnit {
  readonly $name = "rpc";
  readonly router = router;  // oRPC Router

  readonly server: {
    handle(
      request: Request,
      context: RpcContext,
    ): Promise<{ matched: boolean; response: Response | undefined }>;
    router: RpcRouter;
  };

  async $prepare(): Promise<void>;
  async $destroy(): Promise<void>;
}
```

## RPC Context

The RPC context provides `db` and `pubsub` to procedures at request time:

```ts
interface RpcContext {
  db: NodePgDatabase<Record<string, never>>;
  pubsub: PubSubUnit;
}
```

<Callout type="warn">
  The `RpcUnit` constructor accepts `{ auth, db, logs, pubsub }` as deps but does **not** use them at construction time. The `RpcContext` is passed at request time via `handle()`, not injected at construction.
</Callout>

## Handling Requests

```ts
const result = await framework.rpc.server.handle(request, {
  db: framework.db.db,
  pubsub: framework.pubsub,
});

if (result.matched) {
  return result.response;
}
// If not matched, fall through to other handlers
```

## Defining Procedures

Procedures are defined using oRPC's `os` base with Zod validation:

```ts
import { os } from "../base";
import { z } from "zod";

export const echo = os
  .input(z.object({ message: z.string() }))
  .handler(async ({ input }) => {
    return { echo: input.message };
  });
```

## Router Structure

The router is a nested object of procedures:

```ts
// router.ts
export const router = {
  echo: echoProcedure,
  health: {
    check: healthCheckProcedure,
  },
};
```

## Request Matching

The `handle()` method checks if the request URL starts with the configured `prefix`. If it matches, the oRPC handler processes the request. If not, it returns `{ matched: false, response: undefined }` so the caller can fall through to other handlers.

## Base Handler

The `os` base is created with the `RpcContext` type:

```ts
import { os } from "./base";
// os = os.$context<RpcContext>()
```

## Dependency

Receives `{ auth, db, logs, pubsub }` via constructor injection, though these are not used by the unit itself at construction time. The actual context (`RpcContext`) is provided at request time.
