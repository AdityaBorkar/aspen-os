The `KvStoreUnit` (`$name: "kv-store"`) provides a Redis-like key-value API backed by a Postgres `UNLOGGED` table. It is a **core unit** — required in `FrameworkConfig`, not optional.

## Configuration

```ts
interface KvStoreConfig {
  defaultTtl?: number;     // default: 3600 (seconds)
  keyPrefix?: string;      // default: "" (no prefix)
}
```

### Example

```ts
const kvConfig: KvStoreConfig = {
  defaultTtl: 1800,  // 30 minutes
  keyPrefix: "recruiter",
};
```

## Public API

```ts
class KvStoreUnit {
  readonly $name = "kv-store";
  readonly db_schema: typeof db_schema;

  // Basic operations
  async get<T = unknown>(key: string): Promise<T | null>;
  async set(key: string, value: unknown, ttl?: number): Promise<void>;
  async del(key: string): Promise<void>;
  async exists(key: string): Promise<boolean>;

  // Counter operations
  async increment(key: string, amount?: number): Promise<number>;
  async decrement(key: string, amount?: number): Promise<number>;

  // Cache-aside pattern
  async getOrSet<T = unknown>(
    key: string,
    factory: () => Promise<T>,
    ttl?: number,
  ): Promise<T>;

  // Bulk operations
  async clear(pattern?: string): Promise<void>;

  async $prepare(): Promise<void>;
  async $destroy(): Promise<void>;
}
```

## Features

### Basic Get/Set

```ts
// Set a value (uses defaultTtl if ttl not specified)
await framework.kvStore.set("user:123:preferences", {
  theme: "dark",
  language: "en",
});

// Get a value (returns null if expired or not found)
const prefs = await framework.kvStore.get("user:123:preferences");

// Delete a value
await framework.kvStore.del("user:123:preferences");

// Check existence
const exists = await framework.kvStore.exists("user:123:preferences");
```

### TTL (Time-to-Live)

```ts
// Custom TTL (overrides defaultTtl)
await framework.kvStore.set("otp:123", "123456", 300);  // 5 minutes

// TTL of 0 or negative means no expiration
await framework.kvStore.set("config:settings", config, 0);  // never expires
```

<Callout type="info">
  Expired entries are **lazily evicted** on read — when `get()` encounters an expired entry, it deletes it and returns `null`.
</Callout>

### Counter Operations

```ts
// Increment (default by 1)
await framework.kvStore.increment("visits:page:home");      // → 1
await framework.kvStore.increment("visits:page:home", 5);   // → 6

// Decrement
await framework.kvStore.decrement("credits:user:123");     // → -1
await framework.kvStore.decrement("credits:user:123", 10); // → -11
```

### Cache-Aside Pattern

The `getOrSet` method implements the cache-aside pattern — if the key exists, return the cached value; otherwise, call the factory, cache the result, and return it:

```ts
const dashboard = await framework.kvStore.getOrSet(
  "dashboard:org:123",
  async () => {
    // This only runs on cache miss
    return await computeExpensiveDashboard("org-123");
  },
  3600,  // cache for 1 hour
);
```

### Pattern-Based Clearing

Clear keys matching a glob pattern:

```ts
// Clear all keys matching a pattern
await framework.kvStore.clear("user:123:*");

// Clear all keys with the configured prefix
await framework.kvStore.clear();
```

<Callout type="warn">
  The `clear()` method uses a cursor-based scan to avoid loading all keys into memory. It internally translates `*` to `%` and `?` to `_` for SQL `LIKE` matching.
</Callout>

## Key Prefixing

If `keyPrefix` is configured, all keys are automatically prefixed:

```ts
// Config: { keyPrefix: "recruiter" }
// Set key: "user:123"
// Actual DB key: "recruiter:user:123"
```

## Value Serialization

- String values are stored as-is
- Non-string values are JSON-serialized on `set()` and JSON-parsed on `get()`
- If parsing fails, the raw value is returned

## KV Store Table

The `kv_store` table is an `UNLOGGED TABLE` — no Write-Ahead Log (WAL) for performance. This means data may be lost on Postgres crash (by design — cache semantics).

| Column | Type | Description |
|---|---|---|
| `key` | text (PK) | Cache key (may include prefix) |
| `value` | text | JSON-serialized or raw string |
| `expires_at` | timestamptz | TTL expiration (nullable = no expiry) |
| `updated_at` | timestamptz | Last update timestamp |

## Dependency

Receives `{ db: DatabaseUnit }` via constructor injection. Uses the Drizzle instance for all SQL operations.
