The `LogUnit` (`$name: "logs"`) provides structured logging with a buffered write strategy — log entries are accumulated in memory and flushed to Postgres in batches.

## Configuration

```ts title="src/server/log/types.ts"
interface LogConfig {
  defaultLevel?: LogLevel;  // default: "info"
  serviceName?: string;     // default: "app"
}

type LogLevel = "debug" | "info" | "warn" | "error" | "fatal";
```

### Example

```ts
const logConfig: LogConfig = {
  defaultLevel: "info",
  serviceName: "recruiter-api",
};
```

## Public API

```ts title="src/server/log/index.ts"
class LogUnit {
  readonly $name = "logs";

  // Logging methods
  debug(message: string, metadata?: Record<string, unknown>): void;
  info(message: string, metadata?: Record<string, unknown>): void;
  warn(message: string, metadata?: Record<string, unknown>): void;
  error(message: string, error?: Error, metadata?: Record<string, unknown>): void;
  fatal(message: string, error?: Error, metadata?: Record<string, unknown>): void;
  log(level: LogLevel, message: string, metadata?: Record<string, unknown>): void;

  // Child logger with merged context
  child(context: Record<string, unknown>): ChildLogger;

  // Query API
  async query(filter: LogQuery): Promise<LogEntry[]>;
  async getStats(service?, startTime?, endTime?): Promise<LogStats>;

  async $prepare(): Promise<void>;
  async $cleanup(): Promise<void>;
}
```

## Features

### Buffered Writing

Logs are accumulated in a buffer (capacity: 100 entries) and flushed to Postgres every 5 seconds. On destroy, the buffer is drained to ensure no logs are lost.

```ts
framework.logs.info("User created", { userId: "123" });
framework.logs.error("Database error", new Error("Connection refused"), { service: "db" });
```

### Child Loggers

Create scoped loggers that merge context metadata into every log entry:

```ts
const requestLogger = framework.logs.child({
  requestId: "req-abc-123",
  userId: "user-456",
});

requestLogger.info("Processing request");    // includes requestId + userId
requestLogger.error("Validation failed", err); // includes requestId + userId + error
```

### Level Filtering

Log levels are prioritized: `debug(0) < info(1) < warn(2) < error(3) < fatal(4)`. Messages below `defaultLevel` are silently dropped.

```ts
// If defaultLevel is "info", debug messages are not logged
framework.logs.debug("This won't be recorded");
framework.logs.info("This will be recorded");
```

### Query API

Query stored log entries with filters:

```ts
const entries = await framework.logs.query({
  level: "error",
  service: "recruiter-api",
  startTime: new Date("2024-01-01"),
  endTime: new Date(),
  userId: "user-123",
  traceId: "trace-abc",
  search: "database",
  limit: 50,
  offset: 0,
});
```

<Callout type="info">
  Both `query()` and `getStats()` automatically scope results to the current `tenantId` from the request context. You don't need to pass `tenantId` manually.
</Callout>

### Log Stats

Get aggregate statistics:

```ts
const stats = await framework.logs.getStats(
  "recruiter-api",
  new Date("2024-01-01"),
  new Date(),
);

// stats = {
//   total: 15234,
//   byLevel: { debug: 5000, info: 8000, warn: 1500, error: 600, fatal: 34 },
//   errorRate: 0.0415
// }
```

## Log Entry Structure

```ts title="src/server/log/types.ts"
interface LogEntry {
  id: string;
  level: LogLevel;
  message: string;
  service: string;
  timestamp: Date;
  metadata?: Record<string, unknown>;
  error?: { name: string; message: string; stack?: string };
  traceId?: string;      // populated from metadata, if provided
  spanId?: string;       // populated from metadata, if provided
  tenantId?: string;     // auto-scoped from request context
  userId?: string;
  requestId?: string;
  duration?: number;
}
```

## Log Table

Logs are stored in the `logs` table with the following columns:

| Column | Type | Description |
|---|---|---|
| `id` | text (PK) | UUID |
| `level` | text | `debug` / `info` / `warn` / `error` / `fatal` |
| `message` | text | Log message |
| `service` | text | Service name |
| `timestamp` | timestamptz | When the log was created |
| `metadata` | jsonb | Additional metadata |
| `error_name` | text | Error name (if applicable) |
| `error_message` | text | Error message |
| `error_stack` | text | Error stack trace |
| `trace_id` | text | Trace ID (from metadata, if provided) |
| `span_id` | text | Span ID (from metadata, if provided) |
| `tenant_id` | text | Tenant scope (default: `"default"`) |
| `user_id` | text | Associated user |
| `request_id` | text | Associated request |
| `duration_ms` | integer | Duration in milliseconds |

## Dependency

Receives `{ db: DatabaseUnit }` via constructor injection. Uses the Drizzle instance for inserting buffered log entries and querying.
