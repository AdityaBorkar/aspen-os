The `ComplianceModule` (`name: "compliance"`) manages compliance documents through their full lifecycle — creation, submission, verification, expiry, renewal, and archival — with obligation templates, automated reminders, and audit trails.

## Configuration

```ts
type ComplianceModuleConfig = {
  country: "INDIA";
  dashboardCacheTtl?: number;             // KV cache TTL for dashboard
  defaultEscalationDays?: number[];        // default: [1, 7, 30]
  defaultReminderDays?: number[];          // default: [90, 60, 30, 7] for expiry, [30, 15, 7, 1] for due
};
```

### Example

```ts
import { ComplianceModule } from "@aspen-os/compliance";

const compliance = ComplianceModule.create({
  country: "INDIA",
  dashboardCacheTtl: 600,
  defaultReminderDays: [90, 60, 30, 7],
  defaultEscalationDays: [1, 7, 30],
});

const framework = Framework.create(config, { compliance });
```

## Features

- **Document lifecycle** — create, submit, verify, reject, complete, renew, archive, snooze
- **Obligation templates** — recurring compliance obligations with auto-generation
- **Verification rules** — role-based reviewer assignment and verification workflows
- **Audit trail** — full audit history for every compliance action
- **Reminder engine** — automated expiry and due-date reminders via PubSub
- **Escalation** — multi-level escalation for overdue documents
- **Dashboard** — cached summary with health score, by-category/status/branch/module breakdowns
- **Event bridge** — subscribes to organization events and generates compliance documents
- **5 scheduled jobs** — daily expiry scan, escalation, status transition, obligation generation, weekly summary

## Workflows

| Workflow | Accessor | Methods |
|---|---|---|
| DocumentWorkflow | `framework.compliance.documents` | `create`, `update`, `submit`, `verify`, `reject`, `complete`, `renew`, `archive`, `snooze`, `assignReviewer`, `uploadAttachment`, `get`, `list`, `getRenewalChain` |
| ObligationWorkflow | `framework.compliance.obligations` | `create`, `update`, `activate`, `deactivate`, `get`, `list`, `previewNextPeriod` |
| VerificationWorkflow | `framework.compliance.verification` | `createRule`, `updateRule`, `getRules`, `assignReviewer` |
| AuditWorkflow | `framework.compliance.audit` | `log`, `getTrail`, `getEntityHistory` |
| DashboardWorkflow | `framework.compliance.dashboard` | `getSummary`, `getTimeline`, `getByCategory`, `getByBranch` |

## Services

| Service | Purpose |
|---|---|
| `ReminderEngine` | Registers pubsub handlers and cron schedules for expiry/due-date reminders |
| `ObligationGenerator` | Listens to obligation schedules and auto-generates compliance documents |
| `EventBridge` | Subscribes to external module events (e.g., organization) and creates compliance documents |
| `StatusDerivation` | Derives verification status from dates and renewal state |

## Database Tables

4 tables: `compliance_document`, `compliance_obligation`, `compliance_verification_rule`, `compliance_audit_entry`.
