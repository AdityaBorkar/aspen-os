The Tasks module manages projects, tasks, statuses, time tracking, comments, attachments, reminders, automation rules, saved views, and dependency graphs.

## Module Configuration

```ts
import { TaskModule } from "@aspen-os/tasks";

const tasks = TaskModule.create({
  enableNotifications: true,
});
```

| Option | Type | Default | Description |
|---|---|---|---|
| `enableNotifications` | `boolean?` | `false` | Instantiates `NotificationBridge` for pubsub event publishing |

## Initialization

```ts title="src/index.ts"
initialize(units: { db: DatabaseUnit; pubsub: PubSubUnit }): void
```

Receives `db` and `pubsub` units. Instantiates all 11 workflows, the `ReportService`, and optionally the `NotificationBridge` (when `enableNotifications` is set). The `ReminderWorkflow` receives the notification bridge instance.

## Accessing Workflows

```ts
platform.tasks.projects       // → ProjectWorkflow
platform.tasks.tasks           // → TaskWorkflow
platform.tasks.statuses        // → StatusWorkflow
platform.tasks.taskTypes       // → TaskTypeWorkflow
platform.tasks.timeEntries     // → TimeEntryWorkflow
platform.tasks.comments        // → CommentWorkflow
platform.tasks.links           // → LinkWorkflow
platform.tasks.collaboration   // → CollaborationWorkflow
platform.tasks.reminders       // → ReminderWorkflow
platform.tasks.automation      // → AutomationWorkflow
platform.tasks.views           // → ViewWorkflow
```

## Accessing Services

```ts
platform.tasks.reports         // → ReportService
```

<Callout type="warn">
  Accessing any workflow or service getter before `initialize()` throws: `"Tasks module not initialized. Call initialize() after platform.initialize()."`
</Callout>
