The Tasks module defines 17 tables and 8 enums using Drizzle ORM.

## Enums

| Enum | Values |
|---|---|
| `task_priority` | `urgent`, `high`, `medium`, `low`, `none` |
| `task_link_type` | `blocks`, `blocked_by`, `related_to`, `duplicates`, `caused_by`, `split_from` |
| `project_status` | `active`, `archived`, `paused` |
| `project_member_role` | `admin`, `member`, `viewer` |
| `status_category` | `backlog`, `unstarted`, `started`, `completed`, `cancelled` |
| `saved_view_type` | `list`, `board`, `calendar`, `timeline` |
| `reminder_type` | `due_date`, `custom`, `overdue` |
| `automation_trigger` | `status_change`, `assignment_change`, `due_date_passed`, `task_created`, `task_updated` |

## Tables

### `task_project`

Top-level project container with a unique key used for task numbering.

| Column | Type | Description |
|---|---|---|
| `id` | text (PK) | UUID |
| `name` | text | Project name |
| `key` | text (unique) | 2–10 uppercase letters, used for task numbers (e.g. `PROJ-1`) |
| `description` | text | Description (nullable) |
| `lead_id` | text | Project lead user ID |
| `status` | project_status | Defaults to `active` |
| `start_date` | timestamptz | Start date (nullable) |
| `target_date` | timestamptz | Target date (nullable) |
| `default_task_type_id` | text (FK → task_type) | Default task type (nullable) |
| `task_counter` | integer | Monotonically incremented per task created, defaults to 0 |
| `created_at` | timestamptz | Creation timestamp |
| `updated_at` | timestamptz | Last update timestamp |

### `task_project_member`

Project membership with role-based access.

| Column | Type | Description |
|---|---|---|
| `project_id` | text (FK → task_project) | Project ID |
| `user_id` | text | User ID |
| `role` | project_member_role | Defaults to `member` |
| `joined_at` | timestamptz | Join timestamp |

Unique index on `(project_id, user_id)`.

### `task_type`

Task type definitions (e.g. Bug, Feature, Epic), project-scoped or global.

| Column | Type | Description |
|---|---|---|
| `id` | text (PK) | UUID |
| `name` | text | Type name |
| `color` | text | Hex color (nullable) |
| `icon` | text | Icon identifier (nullable) |
| `is_default` | boolean | Defaults to `false` |
| `project_id` | text (FK → task_project) | Project scope (nullable = global) |

### `task_status`

Task statuses with category classification and sort ordering.

| Column | Type | Description |
|---|---|---|
| `id` | text (PK) | UUID |
| `name` | text | Status name |
| `category` | status_category | Workflow category |
| `color` | text | Hex color (nullable) |
| `is_default` | boolean | Defaults to `false` |
| `is_resolved` | boolean | Whether this status marks task completion, defaults to `false` |
| `sort_order` | integer | Display order, defaults to 0 |
| `project_id` | text (FK → task_project) | Project scope (nullable = global) |

### `task_status_transition`

Optional transition rules between statuses.

| Column | Type | Description |
|---|---|---|
| `id` | text (PK) | UUID |
| `project_id` | text (FK → task_project) | Project ID |
| `from_status_id` | text (FK → task_status) | Source status |
| `to_status_id` | text (FK → task_status) | Target status |
| `requires_comment` | boolean | Defaults to `false` |
| `requires_role` | text | Required role for transition (nullable) |

Unique index on `(from_status_id, to_status_id, project_id)`.

### `task_label_def`

Label definitions, project-scoped or global.

| Column | Type | Description |
|---|---|---|
| `id` | text (PK) | UUID |
| `name` | text | Label name |
| `color` | text | Hex color (nullable) |
| `project_id` | text (FK → task_project) | Project scope (nullable = global) |

### `task`

| Column | Type | Description |
|---|---|---|

| `status_id` | text (FK → task_status) | Status ID |
