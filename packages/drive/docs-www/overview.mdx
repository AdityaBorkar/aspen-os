The Drive module manages files and folders with versioning, sharing, public links, labels, trash, and search across 6 workflows and 5 internal services.

## Module Configuration

```ts
export type DriveModuleConfig = {
  allowedContentTypes?: string[];        // empty = all types allowed
  defaultDownloadLinkExpiry?: number;    // seconds
  maxDownloadLinkExpiry?: number;        // seconds
  maxFileSize?: number;                  // bytes
  maxNestingDepth?: number;              // folder hierarchy depth
  maxVersions?: number;                 // retained file versions
  trashRetentionDays?: number;          // auto-purge threshold
};
```

```ts
import { DriveModule } from "@aspen-os/drive";

const drive = DriveModule.create({
  allowedContentTypes: ["image/png", "application/pdf"],
  maxFileSize: 2 * 1024 * 1024 * 1024,  // 2 GB
  maxNestingDepth: 15,
  maxVersions: 5,
  trashRetentionDays: 14,
});
```

| Option | Type | Default | Description |
|---|---|---|---|
| `allowedContentTypes` | `string[]?` | `[]` | MIME type allowlist; empty permits all |
| `defaultDownloadLinkExpiry` | `number?` | `3600` | Default signed URL TTL in seconds |
| `maxDownloadLinkExpiry` | `number?` | `604800` | Maximum signed URL TTL in seconds (7 days) |
| `maxFileSize` | `number?` | `5368709120` | Maximum upload size in bytes (5 GB) |
| `maxNestingDepth` | `number?` | `20` | Maximum folder hierarchy depth |
| `maxVersions` | `number?` | `10` | Retained file versions before pruning |
| `trashRetentionDays` | `number?` | `30` | Days before trashed items are auto-purged |

## Prepare Phase

`prepare()` subscribes to the `drive:auto-purge` pubsub topic and schedules it with a daily cron (`0 3 * * *`). The handler calls `TrashWorkflow.purgeExpired()` to delete trashed items older than `trashRetentionDays`.

## Accessing Workflows

```ts
framework.drive.folders      // → FolderWorkflow
framework.drive.files        // → FileWorkflow
framework.drive.labels       // → LabelWorkflow
framework.drive.shares       // → ShareWorkflow
framework.drive.publicLinks  // → PublicLinkWorkflow
framework.drive.trash        // → TrashWorkflow
```

## Accessing Services

```ts
framework.drive.search       // → SearchService
framework.drive.archive      // → ArchiveService
framework.drive.access       // → AccessService
framework.drive.paths        // → PathService
```

<Callout type="warn">
  Accessing any workflow or service getter before `initialize()` throws `Error("Drive module not initialized. Call initialize() after Framework.create().")`.
