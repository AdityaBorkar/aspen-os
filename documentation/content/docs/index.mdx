The Aspen Framework is a TypeScript business framework that provides a set of composable **units** (infrastructure services) and **modules** (domain logic) wired together via a single `Framework` entry point.

<Callout type="info">
  Built on Bun, Drizzle ORM, better-auth, pg-boss, oRPC, and S3-compatible storage.
</Callout>

## Quick Start

```ts
import { Framework } from "@aspen-os/framework/server";
import { OrganizationModule } from "@aspen-os/organization";

const framework = Framework.create(config, {
  organization: OrganizationModule.create({ country: "INDIA" }),
});

await framework.prepare();

await framework.run(async () => {
  const { db, pubsub } = getContext();
  // db: NodePgDatabase (drizzle instance)
  // pubsub: PubSubUnit
});

await framework.destroy();
```

## Three Entry Surfaces

The framework package (`@aspen-os/framework`) exposes two subpaths:

| Subpath | Environment | Purpose |
|---|---|---|
| `./server` | Node/Bun | `Framework` class, all unit classes, config types |
| `./client` | Browser | Client `Framework` class, `createAccessControl`, client config types |

The root `.` export re-exports types and `createAccessControl` (works via Bun workspace resolution).
